If you were to describe semantic HTML to the next cohort of students, what would you say?
Elements such as <header>, <footer> and <article> are all considered semantic because they 
accurately describe the purpose of the element and the type of content that is inside them.


Name two big differences between display: block; and display: inline;.
Block

Takes up the full width available, with a new line before and after (display:block;)

Inline

Takes up only as much width as it needs, and does not force new lines (display:inline;)
